# 📌 RESUMO COMPLETO DO QUE FOI FEITO NO PROJETO (HOJE)

## 1️⃣ Objetivo do projeto

Criar um **website com dashboards de gastos energéticos**, usando:

* dados recolhidos por **Shelly**
* uma **base de dados MySQL**
* um **sistema de login**
* frontend separado do backend

Este modelo permite no futuro:

* crescer (mais gráficos, utilizadores)
* colocar online sem reescrever tudo
* ligar apps móveis ou outros sistemas

---

## 5️⃣ Backend (`main.py`) — o que faz

### 🔹 Criação da API

```python
app = FastAPI()
```

### 🔹 Ativação do CORS

Permite que o browser comunique com o backend mesmo sendo ficheiros locais.

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)
```

Sem isto o browser bloqueia pedidos (`OPTIONS 405`).

---

### 🔹 Ligação à base de dados MySQL

```python
import mysql.connector

db = mysql.connector.connect(
    host="localhost",
    user="energia_user",
    password="energia_pass",
    database="energia"
)
```

---

### 🔹 Endpoint de login

```python
@app.post("/login")
def login(data: LoginData):
```

* Recebe `username` e `password`
* Faz hash da password
* Verifica na base de dados
* Responde:

  * sucesso → `200 OK`
  * erro → `401 Login inválido`

---

## 6️⃣ Segurança da password

### O problema inicial

* Password estava em texto simples na base de dados
* Backend usa hash SHA-256

### Solução aplicada

Atualização da password no MySQL:

```sql
UPDATE users
SET password_hash = SHA2('1234', 256)
WHERE username = 'tomas';
```

👉 Backend e base de dados passaram a falar a mesma “língua”.

## 7️⃣ Frontend (`login.html`) — o que faz

### 🔹 Formulário HTML

Recolhe:

* username
* password

---

### 🔹 JavaScript com `fetch`

```javascript
fetch("http://127.0.0.1:8000/login", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ username, password })
})
```

* Envia dados ao backend
* Recebe resposta
* Decide:

  * login válido → redireciona para `index.html`
  * login inválido → mostra erro

---

## 8️⃣ Como o backend é iniciado

No terminal (dentro da pasta `backend`):

```bash
python -m uvicorn main:app --reload
```

* `uvicorn` é o servidor
* `--reload` reinicia automaticamente quando mudas código
* Substitui:

  * Tomcat
  * Apache do XAMPP

Enquanto o backend estiver ligado:

* o site funciona
* a API responde
